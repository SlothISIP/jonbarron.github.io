Overview

The AI Digital Twin Factory System is an integrated software architecture for smart manufacturing. It combines vision AI, digital twin simulation, predictive maintenance, and production scheduling to enable real-time monitoring and optimization of manufacturing processes.

System Architecture


The microservices architecture centers around an API Gateway that orchestrates five core modules: Vision AI for real-time inspection, Digital Twin for SimPy-based simulation, Predictive Maintenance for failure prediction, Production Scheduling for AI optimization, and Real-time Monitoring through a React dashboard.

Core Modules

  • Vision AI Module: Real-time visual inspection for defect detection
  • Digital Twin Module: SimPy-based process simulation for virtual factory modeling
  • Predictive Maintenance: Equipment health monitoring and failure prediction
  • Production Scheduling: AI-optimized workflow and resource allocation
  • Real-time Monitoring: Dashboard for live factory status visualization

Digital Twin Simulation

The digital twin module uses SimPy (Simulation in Python) to create a virtual replica of the manufacturing process. Key features include:

  • Discrete-event simulation of production lines
  • Equipment state modeling and failure scenarios
  • Process bottleneck identification
  • What-if analysis for process optimization

3D Factory Simulation


The 3D simulation environment visualizes the entire production floor in real-time. Machine states are color-coded (green = running, yellow = idle, red = down) with live OEE metrics displayed above each workstation. The simulation tracks material flow through conveyor systems and identifies bottlenecks in the production process.

Real-time Monitoring Dashboard


The React-based dashboard provides comprehensive factory monitoring with real-time KPI cards showing OEE, throughput, downtime, and quality metrics. The production rate chart displays 24-hour trends, while the machine status table provides instant visibility into each equipment's operational state and recent alerts.

System Architecture

The system follows a microservices architecture:

  1. Data Layer: Time-series database for sensor data, PostgreSQL for operational data
  2. Service Layer: Independent services for each core module
  3. API Gateway: Unified REST API for all services
  4. Frontend: React-based dashboard for visualization

Technology Stack

  • Backend: Python, FastAPI
  • Simulation: SimPy
  • AI/ML: PyTorch, scikit-learn
  • Frontend: React
  • Infrastructure: Docker, Kubernetes
